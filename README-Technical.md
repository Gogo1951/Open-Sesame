# Open Sesame — Technical Reference

This document combines architecture notes and contribution guidance for developers working on Open Sesame. For end-user documentation, see [README.md](https://github.com/Gogo1951/Open-Sesame/blob/main/README.md).

The addon namespace is `OS` (the second return of `...`) rather than the conventional `ns`. Every file starts with `local ADDON_NAME, OS = ...`.

## Architecture

### Event Loop

Core.lua owns the single event frame. Handlers live in the `EventHandlers` table and are dispatched by event name; the frame registers every key in that table at file load.

Events the addon listens for:

- **`PLAYER_LOGIN`** — initialize SavedVariables, defaults, minimap button, print welcome.
- **`PLAYER_ENTERING_WORLD`** — on initial login or `/reload`, wait `OS.WORLD_LOAD_DELAY` (8s) for bag/loot APIs to settle, then run a forced scan and ensure auto-loot is on. On zone changes, just enter a 2s "quiet" window to suppress chat noise.
- **`BAG_UPDATE_DELAYED`** / **`BAG_NEW_ITEMS_UPDATED`** — debounced `ScheduleScan()` (the soft form). Fires often; debouncing collapses bursts.
- **`LOOT_READY`** / **`LOOT_OPENED`** — record the loot-window timestamp for the loot-sound gating in `CHAT_MSG_LOOT`.
- **`CHAT_MSG_LOOT`** — pattern-match the localized "You receive item:" prefix. If the looted item is in `AllowedItems` with value `false` (a lockbox), announce that it'll be auto-opened on unlock. Optionally play the rare-loot sound.
- **`PLAYER_REGEN_ENABLED`** / **`UPDATE_STEALTH`** — re-scan once combat ends or stealth drops.
- **`UNIT_SPELLCAST_SUCCEEDED`** — when the local player casts Pick Lock (spell ID 1804), schedule a forced scan after 0.5s so the newly-unlocked container queues immediately.
- **`UI_ERROR_MESSAGE`** — match `ERR_INV_FULL` (by message text or numeric error code), pause the addon, play the race/gender-specific bag-full sound, and rate-limit re-triggers via `OS.BAG_FULL_COOLDOWN` (10s).
- **`{BANK,GOSSIP,MAIL,MERCHANT,TRADE,QUEST}_CLOSED`** + **`PLAYER_INTERACTION_MANAGER_FRAME_HIDE`** — forced re-scan once the player exits a UI interaction.
- **`LOADING_SCREEN_ENABLED` / `_DISABLED`** — extend the quiet window across loading screens so we don't spam status messages mid-zone.

Combat and stealth gating live in `IsSafeToOpen()`. The function bails early on:

- `!OS.isEnabled` or `OS.isPaused`
- `UnitAffectingCombat("player")`
- `IsInteractionActive()` (covers C_PlayerInteractionManager plus legacy `MerchantFrame`/`MailFrame`/`TradeFrame`/`BankFrame`/`GossipFrame`/`QuestFrame`/`StaticPopup1` shown checks)
- Stealth or Shadowmeld (spell ID 20580; checked because `IsStealthed` doesn't return true under Shadowmeld)
- Active cast/channel via `UnitCastingInfo` / `UnitChannelInfo`
- A live `GameTooltip` showing an item — opening would dismiss the tooltip mid-comparison
- Free bag slots below `OS.MIN_FREE_SLOTS` (4)

### Scan → Queue → Open pipeline

Owned entirely by Core.lua. Three phases:

1. **Scan** — `ScheduleScan(force?)` either runs the work immediately (forced) or debounces via `C_Timer.After(OS.SCAN_DEBOUNCE)` (0.5s). Multiple events arriving in a burst collapse to one scan. Pause/resume state flips here based on `ShouldPause()` (asymmetric thresholds: pause at <`MIN_FREE_SLOTS`, resume only at `MIN_FREE_SLOTS + 1` — this hysteresis avoids flapping when a single slot opens and closes).

2. **Queue** — `BuildQueue()` walks bags 0–4, reads each slot's item ID via `SafeFastItemID()` (which falls back to parsing `item:%d+` out of the link if `GetContainerItemID` returns nil), and pushes `(bag, slot, itemId)` triples into a flat array-backed queue. `AllowedItems[itemId] == true` queues unconditionally; `== false` queues only if `IsItemLocked(bag, slot)` returns false (i.e., the lockbox has been picked open).

3. **Open** — `OpenTick()` fires every `OS.OPEN_TICK_INTERVAL` (0.25s). Pops one item, re-checks the slot still holds the same item ID (avoids opening whatever the player just dragged into that slot), plays the open sound, calls `OS.UseContainerItem(bag, slot)`, then schedules a follow-up check 0.25s later: if the item is still there and still allowed, requeue it (some containers take multiple uses to empty). The timer reschedules itself only while the queue has work AND `IsSafeToOpen()` returns true.

The `OS.state.openTimerLive` flag exists because `C_Timer.After` callbacks can't be cancelled — the flag prevents us from double-scheduling when a scan finishes mid-tick.

`IsItemLocked()` uses a hidden scanning tooltip (`OS_ScanTooltip`) anchored to `WorldFrame`. It scans every line for the localized `LOCKED` global. Anchoring to `WorldFrame` keeps it out of `GameTooltip:GetOwner()` checks elsewhere.

### Speedy Loot

Speedy-Loot.lua is a self-contained module with its own event frame listening for `LOOT_READY`. When fired:

1. Bail if `OS.isSpeedyLoot` is false.
2. Compute `shouldAutoLoot = autoLoot ~= IsModifiedClick("AUTOLOOTTOGGLE")` — respects the user's auto-loot CVar AND any in-progress modifier override (Shift to invert).
3. Rate-limit via `OS.LOOT_DELAY` (0.25s) to avoid double-handling on rapid bag-pickup fires.
4. Hide `LootFrame` immediately so the loot UI doesn't flash.
5. Iterate slots in reverse, calling `LootSlot()` on each.

Two important guards before calling `LootSlot()`:

- **Ignore list**: items in `OS.IgnoreItems` (e.g., quality-graded gem sacks) are skipped and re-shown via `LootFrame:Show()`. The user is told once per item per 5s via `OS.state.recentAnnouncements`.
- **Master-looter threshold**: when the player is the master looter (`lootMethod == "master" and mlPartyID == 0`), items at or above `GetLootThreshold()` are skipped and the loot frame is reopened. *This guard exists because* calling `LootSlot()` on a threshold-crossing item as master looter pops `MasterLooterFrame_Show` with no `selectedLootButton`, which then crashes on a nil `colorInfo`. Don't remove this branch.

The auto-loot detection at the top of the file uses an availability-based pick (`if C_CVar.GetCVarBool then ... else ...`) rather than the more idiomatic `(modern() or legacy())`. The latter falls through to the legacy path whenever the modern API legitimately returns `false` (auto-loot off), causing a double call. Keep the `if`-form.

### Item Allowlist

`OS.AllowedItems` in Data.lua is a flat `{ [itemId] = boolean }` map. The boolean has semantic meaning, not just presence:

- **`true`** — open immediately. Clams, gift bags, fishing junk, quest containers, basically anything that doesn't require a key/lockpick.
- **`false`** — requires unlock. Lockboxes and similar. The pipeline only queues these once `IsItemLocked()` returns false (i.e., a Rogue or key has popped them).

`OS.IgnoreItems` is a presence-only `{ [itemId] = true }` map used by Speedy-Loot to skip auto-looting Bind-on-Pickup grab bags that would otherwise prompt the BoP confirmation dialog mid-loot.

Both tables are organized by expansion via comment dividers (`-- 01. World of Warcraft`, `-- 02. Burning Crusade`, `-- 03. Wrath of the Lich King`). The lists were curated by hand, not generated from a SQL dump, so there's no source query to regenerate from.

## Saved Variables

`OpenSesameDB` (account-wide) holds:

- `autoOpen` (boolean, default `true`) — master switch for the auto-open pipeline. Mirrored to `OS.isEnabled` at runtime.
- `speedyLoot` (boolean, default `true`) — toggles the LOOT_READY handler in Speedy-Loot.lua. Mirrored to `OS.isSpeedyLoot`.
- `lootSounds` (boolean, default `false`) — plays a distinct sound on Uncommon+ loot in `CHAT_MSG_LOOT`.
- `showWelcome` (boolean, default `true`) — toggles the login welcome message.
- `minimap` (table) — owned by LibDBIcon; holds the minimap button position. Pass the subtable directly to `LDBIcon:Register`.

There is no per-character DB. All settings are account-scoped.

Initialization runs in `EventHandlers:PLAYER_LOGIN`. Defaults use the `if db.field == nil then db.field = default end` pattern (not `db.field = db.field or default`) because `false` is a valid stored value for these toggles — the `or` form would silently overwrite a deliberately-disabled toggle on every login.

## Adding a New Item to the Allowlist

1. Find the item ID on Wowhead Classic (or `/run print(GetItemInfoInstant(...))` in-game).
2. Open `Data.lua` and locate `OS.AllowedItems`.
3. Add a row in the appropriate expansion block: `[itemId] = true,` for instant-open items, `[itemId] = false,` for lockboxes that need unlocking. End the line with a `-- Item Name` comment for grep-ability.
4. Keep entries alphabetized within the block by item name (matches the rest of the file).
5. If the item is a BoP grab bag that should never auto-loot, add it to `OS.IgnoreItems` with `[itemId] = true,` instead.

No locale changes required — item names are not localized in the data tables; the inline comments are reference-only.

## Adding a New Locale

Copy `Locales/enUS.lua` to `Locales/<locale>.lua`. Drop the `true` argument from the `NewLocale("OpenSesame", "<locale>", true)` call (the `true` flag marks the file as the default fallback — only `enUS.lua` should set it). Translate every string. Add the file to the `.toc` immediately after `Locales/enUS.lua`.

For Spanish, follow the shared-`strings`-table pattern in `Locales/esES.lua` so a single file registers both `esES` and `esMX`. See [Common Pitfalls](#common-pitfalls) for why the standard `if not L then return end` guard breaks esMX.

## Common Pitfalls

- **Master-looter threshold crash in Speedy Loot**: calling `LootSlot()` on an item at or above the loot threshold while the player is the master looter pops `MasterLooterFrame_Show` with no `selectedLootButton` and crashes on a nil `colorInfo`. Speedy-Loot.lua already guards against this — see the `isMasterLooter` / `lootThreshold` check. Don't remove or simplify it.

- **esMX silently falling back to English**: the obvious `local L = NewLocale("OpenSesame", "esES"); if not L then return end` pattern bails the entire file when an esMX client loads it (because `NewLocale("...", "esES")` returns nil for esMX), so the subsequent `NewLocale("...", "esMX")` never runs. Use the shared `strings` table pattern in `Locales/esES.lua` — build the strings into a local table, then register both locales independently.

- **Truthy-fallthrough on CVar APIs**: `(C_CVar.GetCVarBool("autoLootDefault")) or (GetCVar("autoLootDefault") == "1")` looks reasonable but falls through to the legacy call any time the modern API returns `false` (auto-loot off), so the legacy result wins instead. Pick the API by availability (`if C_CVar then ... else ...`), not by truthy result.

- **`C_Timer.After` callbacks can't be cancelled**: the `OS.state.openTimerLive` flag in Core.lua exists to prevent double-scheduling the open tick when a scan completes mid-flight. Keep it consistent if you add new entry points to `OpenTick()`.

- **Reading an item ID right after a bag update**: `GetContainerItemID` can return nil immediately after a bag mutation. Use `SafeFastItemID()` (Core.lua) which falls back to parsing the item link with `item:(%d+)`.

- **Pause hysteresis**: pause threshold is `< MIN_FREE_SLOTS`; resume threshold is `< MIN_FREE_SLOTS + 1`. The asymmetry prevents flapping when a single bag slot opens and closes during normal play. Don't collapse them to a single comparison.

## Contributing

Issues and PRs welcome. Open issues at [github.com/Gogo1951/Open-Sesame/issues](https://github.com/Gogo1951/Open-Sesame/issues). For real-time discussion, the [Discord](https://discord.gg/eh8hKq992Q) is the fastest path.

When filing a bug report, include:

- Game version (Classic Era 1.15.x, TBC Anniversary 2.5.x, etc.) and locale.
- Class and level.
- Repro steps — what was in your bags, what you were doing, what happened vs. what you expected.
- Any chat output prefixed with `Open Sesame //` and any Lua error text.

PR guidelines:

- Keep changes scoped — one feature or fix per PR.
- Match the existing code style (see [Add-on Coding & Style Guide](https://github.com/Gogo1951) — no `=` alignment padding, no abbreviations, locale all user-facing strings).
- If you add or rename a SavedVariables field, write a migration in `PLAYER_LOGIN` rather than overwriting existing values.
- If you change item-list behavior (allowlist semantics, ignore list scope), update this document and `README.md`.
- Update the File Map and Architecture sections of this document if you add a new file or move logic between phases.
